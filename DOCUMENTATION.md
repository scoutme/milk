# 🥛 milk: Local-first Agentic Orchestrator

## 🚀 Overview

**milk** is a sophisticated Command Line Interface (CLI) designed to serve as a **Local-first Agentic Orchestrator**. It intelligently manages and routes conversational prompts between a high-performance local Large Language Model (LLM, such as Gemma 4 via `llama.cpp`) and a powerful external reasoning agent (Claude Code).

The core philosophy of `milk` is to maintain conversational state and context while dynamically deciding whether a task is best handled locally for speed and privacy, or requires the advanced reasoning capabilities of an external agent.

## 🧠 Architecture and Workflow

The intelligence of `milk` lies in its **Decision Chain**, which determines the routing of every prompt:

1.  **Input:** A user prompt is received.
2.  **Explicit Override:** The system first checks for explicit routing flags (e.g., `--escalate` or `--local`).
3.  **Session Context:** If the previous turn was handled by Claude, the system automatically routes the follow-up to Claude to maintain context.
4.  **Rules Layer (The Scorer):** A configurable set of rules (based on token length, keywords, and code block presence) evaluates the prompt.
5.  **Local Classifier:** If the scorer is inconclusive, the local LLM acts as a classifier, using its own reasoning to decide between `local` or `escalate`.
6.  **Routing:** The prompt is routed to the appropriate backend (Local LLM or Claude).

**Escalation Mechanism:** When the local model determines a task is too complex, it calls an internal `escalate_to_claude()` function. Crucially, `milk` reformats the entire conversation history into context for Claude, ensuring Claude can orient itself immediately without needing manual reformulation.

## ✨ Key Features

### 1. Hybrid Intelligence
*   **Local Speed & Privacy:** Tasks that are well-defined or require quick iteration are handled by the local LLM.
*   **External Reasoning Power:** Complex, abstract, or high-level tasks are routed to Claude for superior reasoning.

### 2. Interactive CLI (REPL)
*   **Persistent Sessions:** Users can start a continuous, stateful conversation (`milk`).
*   **Session Management:** Commands like `/new`, `/list`, and `/drop` allow users to manage multiple independent conversational threads.

### 3. Configurable Routing
The routing logic is highly tunable via `~/.milk/config.json`. Users can adjust:
*   `escalate_above_tokens` / `local_below_tokens`
*   `escalate_keywords` / `local_verb_weight`
*   The weighting system used by the classifier.

### 4. Integrated Tooling (Local Agent Capabilities)
The local LLM is not just a chatbot; it is a powerful coding assistant with access to a suite of tools, allowing it to interact with the file system and network:

*   `bash`: Execute shell commands.
*   `grep`: Search file contents by pattern.
*   `read_file`: Read a file with optional offset/limit.
*   `write_file`: Write content to a file.
*   `edit_file`: Perform exact-string replacement in a file.
*   `list_dir`: List directory contents.
*   `http_get`: Fetch content from a URL.

## ⚙️ Technical Requirements & Setup

### Prerequisites
1.  **Go Runtime:** Go 1.21 or newer.
2.  **Local LLM Backend:** A running `llama.cpp` server configured to expose an OpenAI-compatible API (default: `http://localhost:8080`).
3.  **External Agent:** The Claude CLI must be installed and configured for the escalation feature to function.

### Usage Examples

**1. Starting an Interactive Session:**
```bash
milk
```
*(The prompt will show `[local]` initially, indicating the default routing.)*

**2. Forcing Escalation:**
```bash
milk --escalate "Design a robust microservice architecture for a global e-commerce platform."
```

**3. Starting a Named Session:**
```bash
milk --session "ecom_design"
```

For detailed configuration parameters, refer to the structure of `~/.milk/config.json`. This file governs the thresholds and weights used by the routing engine.
